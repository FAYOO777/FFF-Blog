```javascript
// 防抖函数实现
function debounce(fn, delay = 300) {
  let timer = null;
  return function (...args) {
    clearTimeout(timer);
    timer = setTimeout(() => {
      fn.apply(this, args);
    }, delay);
  };
}

const handleResize = debounce(() => {
  console.log("窗口尺寸已变化");
}, 500);

window.addEventListener("resize", handleResize);
```

```jsx
    document.title = `计数：${count}`;
  }, [count]);

  return (
    <div className="counter">

}
```

```python
def word_count(text: str) -> int:
    """统计文章字数（中英文混合）"""
    import re
    # 中文字符
    chinese = len(re.findall(r'[\u4e00-\u9fff]', text))
    # 英文单词
    english = len(re.findall(r'\b[a-zA-Z]+\b', text))
    return chinese + english
```

### 10.2 块级公式

$$
x = \frac{-b \pm \sqrt{b^2 - 4ac}}{2a}
$$

正态分布的概率密度函数：

$$
f(x) = \frac{1}{\sigma\sqrt{2\pi}} \exp\!\left(-\frac{(x-\mu)^2}{2\sigma^2}\right)

$$

\begin{pmatrix} x \\ y \end{pmatrix}
=

$$
P(A \mid B) = \frac{P(B \mid A)\,P(A)}{P(B)}

## 十三、混合排版压力测试

以下段落将多种格式混合使用，以测试复杂场景下的渲染稳定性。

在现代前端开发中，**组件化思维**已经成为构建大型应用的核心范式。以 React 为例，一个典型的组件可能长这样：

```jsx
export function Badge({ label, variant = "default" }) {
  return <span className={`badge badge-${variant}`}>{label}</span>;
}
```

其核心理念是：*单一职责* + *可复用性* + *可测试性*。常见的状态管理方案包括：

1. **useState** — 适合组件局部状态
2. **useReducer** — 适合复杂状态逻辑
3. **Context API** — 适合跨层级共享
4. **Zustand / Jotai** — 轻量级全局状态库
5. **Redux Toolkit** — 企业级复杂状态场景

> 选择状态管理方案时，应遵循"够用就好"的原则——过度设计带来的维护成本，往往高于它解决的问题。[^arch]

[^arch]: 这是关于软件架构的脚注，引自软件工程领域的普遍共识。

相关性能指标参考：

| 指标         | 目标值       | 测量工具              |
| ------------ | ------------ | --------------------- |
| LCP          | ≤ 2.5s       | Lighthouse / CrUX     |
| FID / INP    | ≤ 200ms      | Web Vitals            |
| CLS          | ≤ 0.1        | Lighthouse            |
| TTFB         | ≤ 800ms      | WebPageTest           |

最后，记住公式：$\text{性能} = f(\text{代码质量}, \text{架构设计}, \text{基础设施})$
