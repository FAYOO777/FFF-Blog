## 五、Hysteria2 正式配置：域名 + ACME 证书

设置邮箱变量：

```bash
ACME_EMAIL="你的邮箱"
```

创建 ACME 目录并设置权限：

```bash
mkdir -p /etc/hysteria/acme
chown -R hysteria:hysteria /etc/hysteria/acme
```

如果已有旧配置，先备份：

```bash
cp /etc/hysteria/config.yaml /etc/hysteria/config.yaml.bak.$(date +%F-%H%M%S)
```

生成 Hysteria2 密码：

```bash
HY2PASS=$(openssl rand -base64 24)
echo "$HY2PASS"
```

写入正式配置：

```bash
cat > /etc/hysteria/config.yaml <<'EOF'
listen: :443

acme:
  domains:
    - <YOUR_DOMAIN>
  email: <YOUR_EMAIL>
  ca: letsencrypt
  type: http
  http:
    altPort: 80
  dir: /etc/hysteria/acme

auth:
  type: password
  password: "<HY2_PASSWORD>"

masquerade:
  type: proxy
  proxy:
    url: https://www.bing.com
    rewriteHost: true
EOF
```

修复权限：

```bash
chown hysteria:hysteria /etc/hysteria/config.yaml
chmod 600 /etc/hysteria/config.yaml
```

启动并重启服务：

```bash
systemctl enable --now hysteria-server.service
systemctl restart hysteria-server.service
```

查看状态：

```bash
systemctl status hysteria-server.service --no-pager -l
```

如果日志出现：

```text
certificate obtained successfully
server up and running
```

说明证书申请成功，服务正常。如果出现 `WARN stapling OCSP`，一般可以忽略，不影响连接。

```bash

print()

print("=====================================")
print()
PY
```

## 七、安装 Xray，新增 VLESS Reality TCP 备用协议

### 7.1 为什么要加 Reality

实测结果：

```text
Wi-Fi + Hysteria2：体验正常，4K 可用
蜂窝数据 + Hysteria2：明显慢
蜂窝数据直连：不慢
蜂窝数据 + Reality TCP：速度恢复，4K 可用
```

判断：蜂窝网络对 UDP/Hysteria2 链路不友好，TCP 443 Reality 更适合蜂窝环境。

最终搭配：

```text
Hysteria2：443/udp，适合 Wi-Fi / 宽带
VLESS Reality：443/tcp，适合蜂窝数据 / 备用默认
```

两个协议可以同时使用 443，因为一个是 UDP，一个是 TCP，不冲突。

### 7.3 生成 Reality 参数

生成 UUID：

```bash
cat /proc/sys/kernel/random/uuid
```

生成 X25519 密钥：

```bash
xray x25519
```

新版 Xray 可能输出类似：

```text
PrivateKey: <PRIVATE_KEY>
Password (PublicKey): <PUBLIC_KEY>
Hash32: <HASH32>
```

对应关系：

```text
PrivateKey：服务端 config.json 使用
Password (PublicKey)：客户端 Loon / Mihomo 使用，也就是 Public Key
Hash32：本次配置不用填
```

生成 ShortId：

```bash
openssl rand -hex 8
```

得到 16 位十六进制字符串，例如：

```text
9f3a1b2c4d5e6f70
```

### 7.5 Xray systemd 注意事项

如果 Xray 服务启动了但没有监听 `443/tcp`，检查：

```bash
systemctl cat xray
```

如果服务使用：

```ini
User=nobody
```

可能无法绑定低端口。可以改为 root：

```bash
nano /etc/systemd/system/xray.service
```

将：

```ini
User=nobody
```

改为：

```ini
User=root
```

重载并重启：

```bash
systemctl daemon-reload
systemctl restart xray
systemctl status xray --no-pager -l
```

检查监听：

```bash
ss -lntup | grep xray
```

理想输出：

```text
tcp   LISTEN  *:443  users:(("xray",pid=...,fd=...))
```

再检查双协议：

```bash
ss -lntup | grep ':443'
```

理想输出：

```text
udp   UNCONN  *:443  hysteria
tcp   LISTEN  *:443  xray
```

## 十四、常见问题

### 14.1 Hysteria2 服务启动失败：permission denied

修复：

```bash
chown hysteria:hysteria /etc/hysteria/key.pem /etc/hysteria/cert.pem /etc/hysteria/config.yaml
chmod 600 /etc/hysteria/key.pem
chmod 644 /etc/hysteria/cert.pem
chmod 600 /etc/hysteria/config.yaml
systemctl restart hysteria-server.service
```

当前正式 ACME 配置没有 `/etc/hysteria/server.crt` 或 `/etc/letsencrypt/live/...` 路径，证书由 Hysteria2 内置 ACME 管理在：

```text
/etc/hysteria/acme
```

### 14.2 Xray active 但没监听 443/tcp

检查：

```bash
ss -lntup | grep xray
systemctl cat xray
journalctl -u xray -n 100 --no-pager
```

如果使用 `User=nobody`，尝试改为：

```ini
User=root
```

然后：

```bash
systemctl daemon-reload
systemctl restart xray
ss -lntup | grep ':443'
```

理想状态：

```text
tcp LISTEN *:443 xray
```

### 14.3 Reality 客户端连接失败

重点检查：

```text
UUID 是否和服务端 clients.id 一致
Public Key 是否使用 xray x25519 输出的 Password (PublicKey)
是否错误使用了 Hash32
Short ID 是否使用服务端 shortIds
SNI 是否为 www.bing.com
Flow 是否为 xtls-rprx-vision
端口是否为 443
传输是否为 tcp
```

特别注意：

```text
SNI：www.bing.com
不是：<YOUR_DOMAIN>
```

### 14.4 Loon URL 导入不支持

如果 Loon 不支持直接导入 `vless://`，可以在配置文本 `[Proxy]` 下手动添加：

```ini
InterServer-Reality = VLESS,<YOUR_DOMAIN>,443,"<UUID>",transport=tcp,flow=xtls-rprx-vision,public-key="<PUBLIC_KEY>",short-id=<SHORT_ID>,udp=true,block-quic=false,over-tls=true,sni=www.bing.com
```

### 14.5 蜂窝数据下 HY2 慢

现象：

```text
Wi-Fi + HY2 正常
蜂窝 + HY2 慢
蜂窝直连不慢
```

处理：

```text
优先使用 InterServer-Reality
HY2 保留为 Wi-Fi 或备用
```

不建议继续反复调整 HY2 的 `block-quic`，实测对该问题帮助不明显。

### 14.6 中文配置文件乱码

Sublime 保存配置时建议：

```text
File -> Save with Encoding -> UTF-8
```

给 Loon / iPhone / Linux 用，优先使用 UTF-8，不要保存为 GBK。

## 十七、回滚方案

停用 Reality，不影响 HY2：

```bash
systemctl stop xray
systemctl disable xray
```

确认 HY2 仍在：

```bash
systemctl status hysteria-server.service --no-pager -l
ss -lntup | grep ':443'
```

只要看到：

```text
udp *:443 hysteria
```

Hysteria2 仍然正常。

一般不建议停用 HY2，只保留 Reality。HY2 在 Wi-Fi / 宽带下体验很好，建议保留作为备用。
